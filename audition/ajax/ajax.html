ajax几种请求方式？他们的优缺点？ ​

常用的post,get,delete。不常用copy、head、link等等。

  1. get
  2. post

区别

  1. post比get安全(因为post参数在请求体中。get参数在url上面)
  2. get传输速度比post快 根据传参决定的。(post通过请求体传参，后台通过数据流接收。速度稍微慢一些。而get通过url传参可以直接获取)
  3. post传输文件大理论没有限制 get传输文件小大概7-8k ie4k左右
  4. get获取数据 post上传数据(上传的数据比较多 而且上传数据都是重要数据。所以不论在安全性还是数据量级 post是最好的选择)

什么情况造成跨域？ ​

同源策略限制 不同源会造成跨域。

原因
http://协议不同，http和https
www子域名不同
baidu.com主域名不同
8080端口号不同
www.baidu.comip地址和网址不同

跨域解决方案有哪些？ ​

  1. jsonp 只能解决get跨域(问的最多) 原理：动态创建一个script标签。利用script标签的src属性不受同源策略限制。因为所有的src属性和href属性都不受同源策略限制。可以请求第三方服务器数据内容。

步骤：

  1. 去创建一个script标签
  2. script的src属性设置接口地址
  3. 接口参数,必须要带一个自定义函数名 要不然后台无法返回数据。
  4. 通过定义函数名去接收后台返回数据
js
//去创建一个script标签
var  script = document.createElement("script");
//script的src属性设置接口地址 并带一个callback回调函数名称
script.src = "http://127.0.0.1:8888/index.php?callback=jsonpCallback";
//插入到页面
document.head.appendChild(script);
//通过定义函数名去接收后台返回数据
//在以上代码中，我们定义了一个名为 jsonpCallback 的函数作为回调函数，
//在请求的URL中添加 callback=jsonpCallback 参数，
//使得服务端返回的数据可以作为该函数的参数传递到客户端，并在控制台中输出数据。
function jsonpCallback(data){
    //注意  jsonp返回的数据是json对象可以直接使用
    //ajax  取得数据是json字符串需要转换成json对象才可以使用。
    console.log(data);
}
  1. CORS：跨域资源共享 原理：服务器设置Access-Control-Allow-Origin响应头之后，浏览器将会允许跨域请求
js
// 需要后台设置
Access-Control-Allow-Origin: *              //允许所有域名访问，或者
Access-Control-Allow-Origin: http://a.com   //只允许所有域名访问

限制：浏览器需要支持HTML5，可以支持POST，PUT等方法兼容ie9以上 3. 代理：代理是一种常用的解决跨域请求的方法，它通过在服务器端设置一个代理，将客户端请求发送给目标服务器，并将响应返回给客户端。客户端只与代理服务器交互，而代理服务器则与目标服务器交互。 代理服务器是一种位于客户端和目标服务器之间的中间层服务器。当客户端请求目标服务器的资源时，请求会先发送到代理服务器，代理服务器再将请求发送到目标服务器，然后将响应返回给客户端。通过这种方式，代理服务器可以绕过浏览器的同源策略限制，从而实现跨域请求。

  1. WebSocket：WebSocket是HTML5中引入的一种双向通信协议，它支持跨域请求，可以在不同域名的页面之间进行实时通信。WebSocket是一种长连接，可以持续地发送和接收数据，而不需要每次请求都建立新的连接。 WebSocket协议有以下特点：
  2. 全双工通信：WebSocket允许客户端和服务器之间实现双向通信，可以在任何时候发送和接收数据，不需要等待请求或响应。
  3. 基于消息：WebSocket通信是基于消息的，而不是基于请求和响应的。客户端和服务器之间可以发送和接收任意类型的消息，如文本、二进制数据等。
  4. 高效：WebSocket协议使用单个TCP连接，可以减少网络延迟和带宽占用，从而提高通信的效率。
  5. 跨域支持：与Ajax请求不同，WebSocket通信可以跨域进行，不受同源策略的限制。
  6. 安全性：WebSocket支持使用SSL/TLS加密连接，从而保证通信的安全性。

axios的请求响应拦截 ​

  1. 响应拦截
    • 全局错误处理：你可以在响应拦截器中检查响应的状态码，并根据不同的状态码执行相应的操作。例如，如果响应状态码是401（未授权），你可以执行相应的操作，例如重新登录或跳转到登录页面。
    • 统一的数据处理：你可以在响应拦截器中对响应数据进行处理，例如将响应的数据格式化、转换成特定的数据结构或进行数据解析。
    • 全局Loading状态管理：你可以在请求发送和响应返回的过程中显示和隐藏全局Loading状态。通过在请求开始时显示Loading状态，然后在响应拦截器中隐藏Loading状态，可以提供更好的用户体验。
    • Token刷新：如果你的应用使用了Token进行身份验证，你可以在响应拦截器中检查响应的状态码，并根据特定的状态码来刷新Token。这样可以确保Token始终处于有效状态，并且在过期时自动刷新。
  2. 请求拦截
    • 添加请求头：你可以在请求拦截器中设置请求头，例如添加身份验证的Token添加到请求头、设置请求的内容类型等。这样可以确保每个请求都携带所需的头部信息。
    • 请求参数处理：你可以在请求拦截器中对请求的参数进行处理，例如添加通用的参数、修改参数的格式或加密参数等。这样可以在发送请求之前对参数进行统一处理。
    • 全局Loading状态管理：你可以在请求拦截器中显示全局Loading状态，以提供用户反馈。通过在请求开始时显示Loading状态，然后在请求完成之后隐藏Loading状态，可以使用户了解请求的状态。
    • 请求错误处理：你可以在请求拦截器中检查请求的错误，并根据需要执行相应的错误处理操作。例如，如果请求需要登录权限但用户未登录，你可以在请求拦截器中重定向到登录页面或执行其他操作。
